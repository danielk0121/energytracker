# 에너지트래커 기술 고도화 상세 보고서 (Senior Engineer View)

본 문서는 **에너지트래커** 프로젝트의 Phase 1 프로토타입 분석 결과를 바탕으로, 클린 코드와 성능 최적화, 그리고 Phase 2(실제 OCR 및 클라우드 연동)를 위한 구체적인 작업 목록을 정리한 보고서입니다.

---

## 🧐 현재 상태 분석 및 개선 방향
- **코드 구조**: `index.html`이 900라인을 넘어서며 CSS, JS, HTML이 강하게 결합되어 있어 유지보수가 어려움. -> **모듈화 필요**
- **데이터 처리**: `IndexedDB` 도입으로 저장 공간은 확보했으나, 대량 데이터 렌더링 시 $O(N^2)$ 복잡도의 로직이 산재함. -> **알고리즘 최적화 필요**
- **기능 완성도**: 사진 분석 로직이 Mock 함수 기반임. -> **Tesseract.js 등 실엔진 연동 필요**

---

## 🚀 향후 작업 TODO List

## 1. 코드 아키텍처 및 모듈화 (Code Architecture)
- [ ] 001 **CSS/JS 분리**: `index.html` 내의 스타일과 스크립트를 외부 파일로 분리하여 코드 가독성 및 캐싱 효율 증대.
- [ ] 002 **React/TypeScript 전환**: 복잡한 UI 상태 관리와 타입 안정성을 위해 현대적인 개발 환경으로 마이그레이션.
- [ ] 003 **유틸리티 함수 통합**: 날짜 포맷팅, 숫자 계산 로직 등 중복되는 유틸리티들을 별도 모듈로 공통화.

## 2. 성능 최적화 (Performance Optimization)
- [ ] 004 **목록 렌더링 알고리즘 개선**: `renderUsageList`에서 `findIndex`를 사용하여 데이터를 찾는 로직($O(N^2)$)을 Map 객체 캐싱 기반($O(N)$)으로 변경.
- [ ] 005 **가상 렌더링(Virtual Scrolling) 도입**: 수백 개 이상의 상세 사용 기록 목록을 렌더링할 때 브라우저 부하를 줄이기 위해 화면 노출 영역만 렌더링.
- [ ] 006 **IndexedDB 데이터 압축**: 이미지 데이터를 저장하기 전 WebP 등으로 변환하거나 압축하여 DB 사용량 최적화.

## 3. 핵심 기능 구현 - Phase 2 (Core Features)
- [ ] 007 **Tesseract.js 실제 연동**: 목(Mock) 분석 함수를 실제 OCR 엔진으로 교체하여 이미지 내 숫자 인식 기능 구현.
- [ ] 008 **OCR 인식 보정 로직**: 이전 데이터와의 연속성을 분석하여 오인식된 값(예: 8을 0으로 인식)을 자동 보정하는 필터 구현.
- [ ] 009 **Google Drive API 연동**: 사용자의 개인 클라우드 저장소에 데이터를 백업하고 동기화하는 인터페이스 설계 및 구현.

## 4. UI/UX 및 접근성 (UI/UX Refinement)
- [ ] 010 **차트 인터랙션 고도화**: 드래그 스크롤 시 관성 효과를 추가하고, 데이터 포인트 클릭 시 상세 정보를 보여주는 툴팁 강화.
- [ ] 011 **설정 페이지 기능 실체화**: `settings.html`의 데이터 초기화, 테마 설정 등을 실제 영구 저장소와 연동하여 동작 보장.
- [ ] 012 **접근성(Accessibility) 강화**: 차트 데이터에 대한 ARIA 레이블 추가 및 고대비 테마 지원으로 웹 접근성 준수.

## 5. 안정성 및 테스트 (Stability & Testing)
- [ ] 013 **단위 테스트(Unit Test) 작성**: 전력량 예측 로직 및 데이터 필터링 로직에 대한 테스트 코드 추가 (Jest 등).
- [ ] 014 **E2E 테스트 구축**: 사진 업로드부터 분석, 데이터 반영까지의 핵심 워크플로우를 자동 검증하는 테스트 시나리오 작성.
- [ ] 015 **IndexedDB 마이그레이션 스크립트**: DB 스키마 변경 시 기존 사용자 데이터를 안전하게 보존하고 이전하는 마이그레이션 로직 작성.
