## 🧐 현재 상태 요약
- **구조적 측면**: HTML, CSS, JS 파일이 분리되어 있으나 `script.js`가 약 700라인에 육박하는 모놀리식 구조임.
- **성능 측면**: 차트 렌더링 및 목록 검색 시 데이터 양이 늘어날수록 성능 저하가 우려되는 $O(N^2)$ 로직이 존재함.
- **기능적 측면**: OCR 로직이 Mock 데이터 기반이며, 실제 이미지 분석 엔진(Tesseract.js 등)과의 연동이 시급함.

## 🚀 향후 작업 TODO List

## 1. 코드 구조 개선 및 모듈화 (Architecture & Refactoring)
- [ ] 001 **JS 로직 모듈 분할**: `script.js` 내의 데이터 생성, IndexedDB 관리, UI 렌더링 로직을 각각의 ES 모듈로 분리.
- [ ] 002 **설정값(Constants) 관리**: 차트 색상, 필터 기준, API 설정 등을 별도의 `config.js`로 추출하여 유지보수성 향상.
- [ ] 003 **컴포넌트 기반 렌더링 도입**: 반복되는 차트 항목이나 리스트 아이템을 템플릿 함수나 클래스 기반 컴포넌트로 구조화.

## 2. 알고리즘 및 렌더링 최적화 (Performance Optimization)
- [ ] 004 **데이터 접근 방식 개선**: `findIndex`나 `filter`를 반복 호출하는 대신, 날짜를 키로 하는 Map 객체를 활용하여 데이터 조회 성능을 $O(1)$로 최적화.
- [ ] 005 **차트 렌더링 효율화**: DOM 요소를 매번 생성하는 대신, 변경된 부분만 업데이트하거나 대량 데이터의 경우 Canvas API 도입 검토.
- [ ] 006 **이미지 전처리 파이프라인**: OCR 분석 전 이미지의 대비(Contrast)를 높이거나 흑백화하여 인식률을 높이고 분석 시간을 단축하는 로직 추가.

## 3. 핵심 기능 고도화 - Phase 2 (Core Features)
- [ ] 007 **Tesseract.js 실엔진 통합**: 웹 워커(Web Worker)를 사용하여 UI 블로킹 없이 브라우저에서 직접 OCR을 수행하는 로직 구현.
- [ ] 008 **데이터 무결성 검증**: OCR 결과값이 비정상적(예: 전월보다 급격히 낮음)일 경우 사용자에게 확인을 요청하는 유효성 검사 로직 추가.
- [ ] 009 **사용자 설정 영속화**: 테마 설정 및 기본 필터 옵션을 `localStorage`에 저장하여 재방문 시 사용자 경험 유지.
