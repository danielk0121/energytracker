# 에너지트래커 UI/UX 분석 보고서 (2026.03.13)

본 문서는 '에너지트래커' 프로토타입의 UI/UX 분석 결과를 바탕으로 디자인 및 사용자 경험 개선 제안을 담고 있습니다.

## 1. 디자인 UI 수정 제안
*디자인 철학: 모바일 조작 용이성, 간결함, 애니메이션 최소화*

### 1.1 시각적 계층 구조 강화
- **문제**: 현재 카드(Card) 디자인의 배경색과 테두리가 모바일의 다양한 조명 환경에서 구분이 모호할 수 있습니다.
- **제안**: 카드에 미세한 그림자(Box-shadow)를 추가하거나 배경색과 카드의 대비를 약간 더 높여 시각적 분리감을 강화합니다.

### 1.2 차트 가독성 개선
- **문제**: 차트의 Y축 숫자와 X축 날짜 텍스트 크기가 10px로 매우 작아 가독성이 떨어집니다.
- **제안**: 최소 12px 이상의 폰트 크기를 유지하고, 중요한 지점(최대/최소) 외의 텍스트는 색상을 흐리게 하여 핵심 정보에 집중하게 합니다.

### 1.3 버튼 및 입력 요소 최적화
- **문제**: 필터 버튼 그룹(`filter-group`)의 버튼들이 너무 작아 정밀한 터치가 필요합니다.
- **제안**: 버튼의 패딩을 늘리고 터치 영역(Touch Target)을 최소 44x44px 이상 확보하여 한 손 조작 편의성을 높입니다.

### 1.4 아이콘 시스템 정립
- **문제**: 상단 테마 전환 버튼과 메뉴 버튼이 텍스트/이모지 기반으로 되어 있어 시각적 일관성이 부족합니다.
- **제안**: SVG 아이콘 라이브러리(예: Lucide, Material Icons)를 도입하여 간결하고 전문적인 인상을 줍니다.

## 2. UX 수정 제안
*UX 철학: 좁은 화면 최적화, 친절한 안내 설명 및 예시 포함*

### 2.1 첫 방문자 온보딩 가이드 추가
- **문제**: 앱 실행 시 빈 화면과 샘플 데이터만 보여 사용자가 사진 업로드 기능을 즉시 인지하기 어렵습니다.
- **제안**: 메인 화면 상단에 '시작하기 가이드' 카드를 배치하여, 사진을 찍는 방법과 데이터가 어떻게 분석되는지 친절하게 설명합니다.

### 2.2 사진 분석 상태 피드백 강화
- **문제**: '이미지 분석 중...'이라는 텍스트 외에 진행률을 알 수 있는 시각적 피드백이 부족합니다.
- **제안**: 애니메이션 이펙트를 최소화하되, 진행 바(Progress Bar)나 단계별 상태(Step Indicator)를 표시하여 사용자의 대기 지루함을 해소합니다.

### 2.3 사진 업로드 예시 상세화
- **문제**: 현재 `wallpad_sample.png` 하나만 제공되어 다양한 각도나 환경에서의 사진 인식률에 대한 예시가 부족합니다.
- **제안**: '좋은 예시'와 '나쁜 예시'(너무 어두움, 흔들림 등)를 나란히 보여주어 사용자가 올바른 사진을 찍도록 유도합니다.

### 2.4 설정 항목의 상태 안내
- **문제**: 설정 화면의 '샘플 데이터 사용' 항목이 비활성화(disabled)되어 있으나 그 이유를 알 수 없습니다.
- **제안**: 비활성화된 항목 옆에 '준비 중' 또는 '현재 버전 고정' 등의 툴팁이나 보조 텍스트를 추가하여 친절하게 안내합니다.

## 3. 분석 요약 및 향후 과제
현재 프로토타입은 핵심 기능이 명확하게 구현되어 있으나, 모바일 환경에서의 정교한 터치 조작과 처음 사용하는 사용자를 위한 배려(안내 문구) 부분이 보완된다면 더욱 훌륭한 서비스가 될 것입니다.
