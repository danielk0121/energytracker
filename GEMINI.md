# LLM 규칙
*LLM 은 이 규칙을 숙지하고 반드시 따라야 한다!*

## 작업 규칙
- 1개 작업 단위로 코드를 수정하고, 커밋한다.
  - 1개 작업을 시작할때 main 브랜치에서 새로운 브랜치를 만들어서 작업한다.
  - main 에서 새로운 브랜치를 만들기 전에 fetch 하여 origin 에서 내용을 받아 온 후 진행한다.
  - **작업 완료 후 main에 머지하더라도, 개별 작업 브랜치는 삭제하지 않고 보존합니다.**
  - **작업 완료 후 커밋하기 전에 최상단에 있는 버전 정보(`v-yyMMdd-HHmm-kst`)를 현재 시간값으로 수정 후 커밋한다. 예를들어, 현재시간이 2026-03-12 00:56:30 KST 이면 v-260312-0056-kst 를 사용한다.**
- 모든 문서와 커밋 메세지, 코드 주석 등 한글을 사용한다.
- LLM 아웃풋 응답도 한글을 사용한다.
